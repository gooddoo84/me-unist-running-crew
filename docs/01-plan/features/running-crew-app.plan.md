## 1. 프로젝트 개요

### 1.1 배경
ME UNIST 러닝 크루는 함께 달리는 커뮤니티로, 멤버들의 러닝 기록을 관리하고 월별 목표를 추적할 수 있는 전용 웹 앱이 필요하다.

### 1.2 목표
- 멤버별 러닝 거리 기록 및 월별 자동 누적
- 월별 목표 거리 설정 및 달성률 시각화
- 멤버 간 랭킹 시스템으로 건전한 경쟁 유도
- 모바일 친화적이고 멋진 UI/UX

### 1.3 비목표 (Out of Scope)
- 회원가입/로그인 인증 시스템 (간단한 멤버 선택 방식으로 대체)
- GPS 기반 자동 러닝 트래킹
- 소셜 기능 (댓글, 좋아요 등)
- 네이티브 모바일 앱

### 2.1 기능 요구사항

| ID | 기능 | 설명 | 우선순위 |

| FR-06 | 랭킹 보드 | 월별 달린 거리 기준 멤버 순위 표시 | P0 |

| FR-09 | 대시보드 | 크루 전체 통계 (총 거리, 평균, 목표 달성자 수) | P1 |

| 필드 | 타입 | 설명 |

| runDate | date | 러닝 날짜 |

| 필드 | 타입 | 설명 |

## 5. 핵심 화면 구성

### 5.1 대시보드 (메인 페이지)
- 크루 이름 및 로고
- 현재 월 크루 전체 통계 (총 거리, 멤버 수, 목표 달성자)
- 랭킹 보드 (순위, 이름, 달린 거리, 달성률 프로그레스 바)
- 월 이동 네비게이션

### 5.2 기록 입력 페이지
- 멤버 선택 드롭다운
- 거리 입력 (km)
- 날짜 선택 (기본값: 오늘)
- 제출 버튼

### 5.3 멤버 상세 페이지
- 멤버 프로필 (이름, 이모지)
- 월별 목표 설정/수정
- 이번 달 달성률 원형 프로그레스
- 러닝 기록 히스토리 리스트

## 7. API 엔드포인트 (Server Actions)

| 액션 | 설명 |
|------|------|
| `getMembers()` | 전체 멤버 목록 조회 |
| `addRunningLog(memberId, distance, runDate)` | 러닝 기록 추가 |
| `getMonthlyStats(year, month)` | 월별 멤버별 누적 거리 + 목표 조회 |
| `setMonthlyGoal(memberId, year, month, goalDistance)` | 월별 목표 설정/수정 |
| `getMemberDetail(memberId)` | 멤버 상세 + 기록 히스토리 |
| `getRanking(year, month)` | 월별 랭킹 (거리 내림차순 정렬) |

## 8. 성공 기준 (Success Criteria)

| ID | 기준 | 측정 방법 |
|----|------|-----------|
| SC-01 | 러닝 기록 입력 후 대시보드에 즉시 반영 | 입력 → 대시보드 이동 시 업데이트 확인 |
| SC-02 | 월별 누적 거리가 정확히 합산됨 | 개별 기록 합계 = 월별 총 거리 |
| SC-03 | 목표 대비 달성률이 퍼센트로 정확히 표시 | (달린 거리 / 목표 거리) × 100 |
| SC-04 | 랭킹이 달린 거리 기준 내림차순 정렬 | 상위 멤버가 1등으로 표시 |
| SC-05 | 모바일에서 사용하기 편한 UI | 모바일 뷰포트에서 모든 기능 정상 작동 |
| SC-06 | Vercel에 정상 배포 및 접근 가능 | 배포 URL로 접근 시 정상 동작 |
